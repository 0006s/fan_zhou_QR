# 机器人视觉QR识别系统
这是我们学校实验室组织的一个比赛，需要机器人识别4个二维码（仅一个有 有效信息）

基于 OpenCV WeChatQRCode 的实时多码识别系统，专为机器人比赛场景设计，支持 2×2 矩阵二维码的并行解码、时间平滑防抖、CLAHE 图像增强，以及通过串口与 STM32 下位机通信。

---

## ✨ 特性

- **微信二维码引擎** — 基于 `cv2.wechat_qrcode.WeChatQRCode`，识别率高、鲁棒性强
- **多码同时识别** — 单帧内同时识别画面中所有二维码，适配 2×2 矩阵场景
- **时间平滑防抖** — 连续 N 帧确认才输出，避免误识别和闪烁
- **CLAHE 对比度增强** — 自动增强低光照/模糊画面，提升识别率
- **双版本设计** — 调试版（精美 UI）+ 比赛版（多线程 + 串口通信）
- **串口通信** — 识别结果实时发送给 STM32 下位机，带断线重连

## 🚀 快速开始

### 环境要求

- Python 3.8+
- OpenCV Contrib（需包含 wechat_qrcode 模块）
- 摄像头（USB 或 CSI）

### 安装依赖

```bash
pip install opencv-contrib-python pillow pyserial numpy
```

### 运行调试版

```bash
python testQR_2_old.py
```

- 🟢 绿框 = 已确认的二维码
- 🟡 黄框 = 确认中的二维码
- 按 `Q` 退出

### 运行比赛版

1. 修改 `testQR_3.py` 顶部配置区：

```python
DEBUG_MODE = True          # True=开画面调试, False=比赛模式关闭画面省算力
SERIAL_PORT = 'COM3'       # Windows: COM3 / 树莓派: /dev/ttyUSB0
CAM_INDEX = 1              # 摄像头序号
```

2. 运行：

```bash
python testQR_3.py
```

## 🧠 核心算法

### 识别流程

```
摄像头采集 → CLAHE 增强 → WeChatQRCode 解码 → 时间平滑缓冲池 → 输出结果
```

### 时间平滑机制

- **确认阈值** (`confirm_thresh=3`)：同一内容连续检测到 3 帧才确认输出
- **丢弃阈值** (`drop_thresh=5`)：连续丢失 5 帧才从缓冲池移除
- **计数上限** (`max_count=10`)：防止计数无限增长

### 图像预处理

使用 CLAHE（对比度受限自适应直方图均衡化）在 LAB 色彩空间对 L 通道增强，有效提升低光照和模糊场景下的识别率。

---

## 🔌 串口通信协议（testQR_3）

- **波特率**：115200
- **数据格式**：`{内容}\r\n`（UTF-8 编码）
- **格式验证**：仅发送符合 `颜色,专属编码,核心编码` 格式的数据（如 `Red,1,3`）
- **断线重连**：串口断开后自动尝试重连

## 📋 比赛适配说明

本项目针对 2026 年比赛规则的 **2×2 矩阵二维码** 任务设计：

- 4 个二维码组成 2×2 矩阵，仅 1 个包含有效信息
- 有效二维码位置随机更新
- `testQR_3.py` 的 `is_valid_format()` 方法负责过滤有效信息
